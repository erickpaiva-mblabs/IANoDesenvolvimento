Análise comparativa: desenvolvimento assistido por IA vs. desenvolvimento manual

Case Study: Projeto XPTO

Análise fundamentada em dados reais do projeto XPTO e pesquisas de mercado (GitHub, McKinsey, Stack Overflow).

  • O desenvolvedor manteve foco em decisões de alto nível (arquitetura, UX, design system) enquanto a IA cuidou do boilerplate
  • Pesquisas de mercado corroboram ganhos de 26-55% em produtividade com ferramentas de IA
  • 100% dos arquivos gerados seguem os padrões do projeto com tipagem completa

2. Contexto e Metodologia

Cenário

Desenvolvimento de telas mobile com React Native + TypeScript, seguindo design system no Figma.

  • Componentes reutilizáveis com separação de responsabilidades (styles, types, constants)
  • Integração com design tokens (cores, espaçamentos, tipografia)
  • Ícones SVG customizados e navegação com React Navigation

Ferramentas Utilizadas

Ferramenta Função
Cursor IDE IDE com integração nativa de IA
Claude (Anthropic) Modelo de linguagem para geração de código
Figma (MCP) Design reference integrado via Model Context Protocol

Design Tokens

Uso correto de theme.colors, theme.spacing, theme.borderRadius, theme.fontFamily

Naming Conventions

PascalCase para componentes, camelCase para props, UPPER_SNAKE_CASE para constantes

Imports & Tipagem

Tipagem Completa

Pontos de Atenção

Aspecto Observação
Revisão obrigatória A IA usou bibliotecas incorretas (iconoir-react-native ao invés dos SVGs locais) — corrigido após feedback
Interpretação de requisitos Interpretou "hasNotification" como mostrar/esconder o sino, quando deveria controlar apenas o badge
Valores hardcoded Algumas cores foram hardcoded e precisaram ser adicionadas ao tema
A IA produz código de alta qualidade, mas a revisão humana é indispensável para garantir aderência total aos requisitos.

Benefícios Qualitativos

  • Redução de context switching: a IA interpreta o Figma diretamente (via MCP)
  • Iteração instantânea: correções aplicadas em segundos
  • Menos fadiga cognitiva: foco em "o que construir", não em "como escrever o StyleSheet"
  • Onboarding acelerado: novos devs produtivos mais rapidamente

5. Riscos e Considerações

Risco Severidade Mitigação

6. Conclusão e Recomendações

Resumo dos Ganhos

100% Tipagem completa

Diretrizes de Adoção

  1. Code review obrigatório — toda geração de IA deve passar por revisão humana antes do merge
  2. Padrões documentados — manter regras de projeto atualizadas para que a IA as siga
  3. Treinamento da equipe — investir 1-2 semanas em capacitação de prompts e workflow
  4. Métricas de acompanhamento — monitorar velocity, bugs e satisfação nos primeiros 3 meses

Próximos Passos

  1. Piloto expandido — aplicar IA em mais 2-3 telas e mensurar resultados
  2. Documentação de prompts — criar biblioteca de prompts para padrões recorrentes
  3. Integração com CI/CD — automatizar lint e type-check em código gerado por IA
  4. Avaliação trimestral — revisar ganhos e ajustar estratégia de uso
